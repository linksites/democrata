# Roadmap Democrata

## Norte

Transformar a landing atual em uma base comercial mais consistente, leve e facil de manter, sem perder a identidade premium de burger + pub + barbearia.

## Fase 1 - Correcao e Consistencia

Status desta revisao:

- [x] corrigir referencia quebrada do logo do cabecalho
- [x] alinhar hero com imagem de produto coerente
- [x] reorganizar galeria com melhor uso do acervo
- [x] usar o poster de horarios em contexto mais adequado
- [x] atualizar README e manual
- [ ] consolidar nomes de arquivos de imagem
- [ ] remover duplicados e arquivos sem extensao

## Fase 2 - Performance e Conversao

Prioridade alta:

- [ ] otimizar `assets/logo/nome-democrata.png` e `assets/logo/logoperfil.jpg`
- [ ] definir versoes web canonicas para logo e fotos de cardapio
- [ ] medir cliques em iFood, WhatsApp, telefone e Instagram
- [ ] revisar CTA por secao para reduzir redundancia
- [ ] adicionar SEO basico completo: Open Graph, Twitter Card e canonical

## Fase 3 - Conteudo e Governanca

Prioridade media:

- [ ] centralizar cardapio, precos e horarios em uma fonte unica
- [ ] criar bloco de provas sociais ou depoimentos
- [ ] incluir campanhas sazonais e combos promocionais
- [ ] documentar padrao de nomes para novos assets
- [ ] criar checklist de publicacao com revisao visual obrigatoria

## Fase 4 - Qualidade do Produto

Prioridade media:

- [ ] revisar contraste e acessibilidade com auditoria formal
- [ ] testar navegacao por teclado e foco no lightbox/menu mobile
- [ ] revisar comportamento de reveal em entradas por ancora
- [ ] validar carregamento e legibilidade em redes moveis
- [ ] criar rotina de captura visual desktop/mobile para regressao

## Fase 5 - Evolucao Comercial

Prioridade futura:

- [ ] integrar analytics de conversao por origem
- [ ] destacar horarios de pico, combos e cortes mais procurados
- [ ] preparar versao com campanhas rotativas
- [ ] estudar pagina separada para barbearia ou cardapio expandido

## Criterio de Pronto

Uma proxima iteracao sera considerada solida quando:

- o front usar apenas assets canonicos
- nao houver referencias quebradas nem duplicidades ativas
- as imagens principais estiverem otimizadas para web
- o README refletir o estado real da pagina
- o deploy puder acontecer com `npm run check` e revisao visual simples
